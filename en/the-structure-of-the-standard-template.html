The structure of the Standard template

Abstract

Overview of the structure of the Standard template.

The following sections contain details of all the fields in the Standard template. The Quick Info section, which is always the first section of a template shown to Administrator and Developer users, is a field section in the Standard template. This field section contains item location details, the template's location in the content structure, and various GUIDs related to storage.

Important

All fields in the Standard template begin with double underscores. You access these fields through the properties of the Sitecore.FieldIDs class.

Advanced

The Advanced section associates a data template with its standard values item. It contains the following fields:

  • Standard values - a link to the Standard Values item.

  • Tracking - analytics profile information.

  • Source - designates the cloned item.

Appearance

The Appearance section controls how the Content Editor displays the item. It contains the following fields:

  • Context Menu - a link to a menu displayed when a user right-clicks on an item in the Content Editor’s content tree.

  • Display Name - overrides the item name in the Content Editor.

  • Editor - defines a custom editor for an item.

  • Editors - controls the custom editor tabs that appear in the Content Editor.

  • Hidden - controls whether or not the item is visible in the Content Editor. You can manage the visibility of hidden items by selecting the Hidden Items check box on the View tab in the Content Editor.

  • Icon - controls the icon shown in the content tree and in the item header in the Content Editor.

  • Read Only - controls whether the item appears read-only.

  • Ribbon - provides a technique to customize the Content Editor ribbon.

  • Skin - associates formatting information with the item.

  • Sortorder - used by the system to specify the order in which the fields are displayed.

  • Style - associates formatting information with the item.

  • Subitems Sorting - controls the rule used to sort the children of the item.

  • Thumbnail - associates a thumbnail image with an item.

  • Originator - indicates the branch template used to create the item.

Help

The Help section stores helpful information about the item. It contains the following fields:

  • Help link - the link to the detailed help for this item.

  • Long description - the fly-over help shown when hovering the mouse over an item in the content tree.

  • Short description - the description shown in the item header of the Content Editor.

Layout

The Layout section is used primarily to associate the data template with its renderings. It contains the following fields:

  • Renderings - the presentation components assigned to the item, such as devices, layouts, sublayouts, XSLT Renderings, and placeholder settings.

  • Renderers - the renderers used to display this item.

  • Presets - layout presets available for an item

Note

Managed websites use the Renderings field. Some Sitecore user interface components use the Renderers field.

Lifetime

The Lifetime section is used to hold the publication restrictions for individual versions of this item. It contains the following fields:

  • Valid from - the date from which this version is valid for publication.

  • Valid to - the date to which this version is valid for publication.

  • Hide version - controls whether to hide this version from publication.

Indexing

The Indexing section is used to configure facets and boost settings. It contains the following fields:

  • Boost Value - a float boost value, such as 1.0, 2.3 , or 0.1. A value higher than zero pushes the results nearer the top of the results. A value lower than zero drags the results down nearer to the bottom.

  • Boosting Rules - enables you to set rules for boosting. Boosting lets you push an item higher up in the results.

  • Facets - shows the facets that are displayed when you run a search from this item. Use this field to add or remove the facets that are associated with the item.

Insert Options

The Insert Options section is used to hold a set of insert rules and options. It contains the following fields:

  • Insert Rules - the insert rules.

  • Insert Options - the insert options.

Item Buckets

The Item Buckets section contains settings for items that have been converted to item buckets or made bucketable. It contains the following fields:

  • Bucket Parent Reference - displays the path to the parent item. If you revert an item bucket or bucketable item, this path is used to restore the connection to the parent item.

  • Is Bucket - if this check box is selected it means that the item is a bucket. This is a system field and must not be set manually.

  • Bucketable — select to make an item bucketable.

  • Lock Child Relationship — if an item has children, this field lets you decide whether the children are added to the bucket or remain as children.

  • Default Bucket Query - lets you specify a default query that is run when you open the Search tab, and displays a list of items that match the results of the query.

  • Persistent Bucket Filter - lets you add a query to the search field that you cannot remove so it is always part of the search.

  • Enabled Views - shows which views are enabled for displaying search results. Use this field to add or remove views. There are three views that are globally enabled by default.

  • Default View - a list of views from which you can choose one to set as the default.

  • Quick Actions - actions that you can associate with an item when it appears in the search results. For example, download, preview, or publish.

Publishing

The Publishing section contains publishing restriction information. It contains the following fields:

  • Publish - the date from which this item is valid for publication.

  • Unpublish - the date to which this version is valid for publication.

  • Publishing groups - the valid publishing targets for the item.

  • Never publish - controls whether or not an item is published. This field overrides the Publish and Unpublish fields.

Security

The Security section contains the security settings. It contains the following fields:

  • Owner - the name of the user who is the current owner of the item. For use with the virtual Creator-Owner role.

  • Security - the security access rights for the item.

Statistics

The Statistics section contains the basic creation, revision, and update statistics. It contains the following fields:

  • Created - the date and time the item was created.

  • Created by - the name of the user who created the item.

  • Revision - the revision number, stored as a GUID text string.

  • Updated - the last date and time that the item was updated.

  • Updated by - the name of the user who performed the last update.

Tagging

The Tagging section contains words or phrases used to make associations between items, making them easier for Sitecore search to find. It contains the following field:

  • Semantics - lets you enter a word or phrase to use as a tag.

Tasks

The Tasks section contains reminder information for associated tasks and to store an archive date. It contains the following fields:

  • Archive date - the date the item will be archived.

  • Reminder date - the date when a reminder email message will be sent to the reminder recipients.

  • Reminder recipients - the email address(es) of the recipients of the reminder, separated by a semi colon (;).

  • Reminder text - the text of the reminder email message.

Validation rules

The Validation rules section contains the validation rules. It contains the following fields:

  • Quick Action Bar Validation Rules - the item validation rules displayed in the Quick Action Bar.

  • Validation Button Validation Rules - the item validation rules used by the Validation button in the Content Tree, on the Review tab, in the Proofing group.

  • Validation Bar Validation Rules - the item validation rules displayed in the Validation Bar on the right of the Content Editor.

  • Workflow Validation Rules - the item validation rules that are used in workflow validation.

  • Suppressed Validation Rules - disables global item validation rules.

Workflow

The Workflow section contains the workflow status information.

  • Workflow - the workflow that the item is in.

  • Workflow State - the current workflow state the item is in.

  • Lock - information about whether the item is locked, who locked it, and what date/time it was locked.

  • Default workflow - information about the default workflow for items created from this template.
