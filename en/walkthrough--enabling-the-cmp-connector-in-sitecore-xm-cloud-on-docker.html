Walkthrough: Enabling the CMP connector in Sitecore XM Cloud on Docker

Abstract

How to enable the Sitecore Connect for Content Hub connector to Sitecore Content Management Platform in XM Cloud in a Docker environment.

To use Sitecore Content Management Platform (CMP) in Sitecore XM Cloud on Docker, you must configure and enable the CMP connector.

This walkthrough describes how to:

  • Configure connection strings for CMP

  • Enable the CMP connector

Configure connection strings for CMP

To enable the CMP connector, you must configure a connection to CMP.

To configure a connection:

  1. In Github, clone the project repository you created in Sitecore Cloud Portal.

  2. In your XM Cloud repository folder, open the .env file and enter the following variables:

    Variable

    Value

    Sitecore_CMP_ContentHub

    ClientId={client_id};ClientSecret = {client_secret};UserName={username};Password={password};URI={uri};

    Sitecore_CMP_Service_ServiceBusEntityPathIn

    {Azure Service Bus connection string with incoming topic}

    Sitecore_CMP_Service_ServiceBusSubscription

    {Subscription name}

    Sitecore_CMP_Service_ServiceBusEntityPathOut

    {Azure Service Bus connection string with outcoming topic}

Enable the CMP connector

By default, when you deploy Sitecore XM Cloud the CMP connector is disabled.

To enable the connector:

  1. In your XM Cloud repository folder, open the .env file for editing.

  2. Set the SITECORE_AppSettings_cmpEnabled__define variable to yes.

  3. In the repository folder, open a Windows console. Run this command:

    docker-compose up -d
